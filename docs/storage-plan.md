# 台北之心闖關網頁：Storage Plan

版本日期：2026-07-10

本文件整理 Supabase Storage 實作草案。此階段僅供審核，尚未執行 SQL。

## 1. Bucket

建立獨立 private bucket：

`taipei-challenge-media`

用途：

- 關卡照片。
- 古生物館錄音。
- 第 6 頁與外國朋友合照。

## 2. 路徑格式

所有檔案使用固定格式：

```text
activity-id/team-id/mission-id/file-uuid.ext
```

範例：

```text
00000000-0000-0000-0000-000000000101/00000000-0000-0000-0000-000000000201/00000000-0000-0000-0000-000000000301/photo-uuid.jpg
```

## 3. 上傳限制

| 使用者 | 權限 |
|---|---|
| 學生 | 只能上傳到自己小組路徑 |
| 教師 | 初版不需要替學生上傳媒體 |
| 管理者 | 可管理全部媒體 |

學生送出成果並鎖定後，不可再上傳或刪除媒體。

## 4. 讀取限制

| 使用者 | 權限 |
|---|---|
| 學生 | 只能讀取自己小組媒體 |
| 教師 | 可讀取可見小組媒體 |
| 管理者 | 可讀取全部媒體 |

前端預覽照片與播放錄音時，使用 Signed URL 取得限時連結。

## 5. 刪除限制

| 使用者 | 權限 |
|---|---|
| 學生 | 可刪除自己小組尚未送出的媒體 |
| 教師 | 可退回補件，讓學生重新上傳 |
| 管理者 | 可刪除錯誤媒體 |

## 6. 檔案建議

- 照片上傳前壓縮，最長邊 1600 px。
- 建議每張照片約 1 至 2 MB。
- 錄音建議每段不超過 2 分鐘。
- 錄音格式優先使用瀏覽器支援的 WebM；若裝置支援 M4A 可另行處理。

## 7. Storage Policies

Storage policies 會建立在 `storage.objects` 上，並限制：

- `bucket_id = 'taipei-challenge-media'`
- 透過檔案路徑第二層 `team-id` 判斷是否可存取該小組。
- 搭配 `taipei_challenge.can_access_team(team_id)` 做權限判斷。
