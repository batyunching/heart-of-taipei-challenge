# 台北之心闖關網頁：資料庫與網站實作規劃

版本日期：2026-07-10

本文件為「台北之心・雙語闖關」互動網站的實作前審核文件。此階段只規劃資料庫、權限、儲存、網站架構與部署流程；尚未開始製作網頁程式，也尚未執行 Supabase SQL。

## 1. 專案目標

建立一個手機優先的雙語戶外闖關網站，讓學生小組在活動現場完成場域探索、照片上傳、文字紀錄、錄音任務、外國遊客訪談與成果送出。

網站包含兩個主要使用端：

| 使用端 | 主要用途 |
|---|---|
| 學生端 | 建立隊伍、登入隊伍、完成關卡、上傳照片與錄音、送出成果 |
| 教師後台 | 查看隊伍、通關碼、成果、照片、錄音，管理活動內容與匯出資料 |

正式部署方式：GitHub Pages。

後端服務：Supabase Database、Supabase Auth、Supabase private Storage。

## 2. 最新需求確認

本規劃已納入以下最新修正：

1. 第 6 頁保留「與世界交朋友｜Connecting with the World」。
2. 第 6 頁提供中英並陳的訪談邀請句、4 題訪談問題、合照徵詢句、被拒絕禮貌句與結束感謝句。
3. 第 6 頁每一句英文提供「播放發音 Listen」功能。
4. 第 6 頁學生填寫外國朋友國家，採自由輸入。
5. 第 6 頁學生需上傳合照照片。
6. 第 6 頁學生需勾選是否完成訪談。
7. 第 6 頁不要求英文反思。
8. 不蒐集外國朋友姓名、電話、Email 或社群帳號。
9. 教師後台入口共用密碼為 `tkjhs810`。
10. 教師後台仍搭配 Supabase Auth 與 RLS 控制資料權限。
11. 教師可看到學生隊名與通關碼明文，以便現場協助。
12. 本次活動先建立預設 3 組隊伍，但未來仍可自由增加組數，也允許學生自行建立隊伍。
13. 教師註冊後不需管理者審核。

## 3. Supabase 變更清單

本系統必須使用既有 Supabase Project：

`batyunching's Org / batyunching's project`

不得建立新的 Supabase Project。

預計只新增本系統專用物件，不修改、刪除、重新命名或覆寫既有 `public` schema 內任何資料表、檢視、函式或政策。

| 類型 | 名稱 | 說明 |
|---|---|---|
| schema | `taipei_challenge` | 本網站所有業務資料表均放在此 schema |
| extension | `pgcrypto` | 若尚未啟用，供 UUID、密碼雜湊或安全比對使用 |
| tables | 11 張資料表 | 活動、教師、小組、成員、任務、作答、媒體與紀錄 |
| functions | 約 8 個 | 小組登入、教師註冊、權限判斷、送出、退回補件 |
| triggers | `updated_at` triggers | 自動維護資料更新時間 |
| RLS policies | 每張表啟用 | 學生看自己、教師看授權小組、管理者看全部 |
| Storage bucket | `taipei-challenge-media` | private bucket，儲存照片與錄音 |
| Storage policies | 上傳、讀取、刪除政策 | 依小組、教師、管理者權限限制 |
| seed data | 本次活動預設資料 | 預設活動、3 組隊伍、頁面、任務與句型 |

需要手動確認或設定的項目：

1. 將 `taipei_challenge` 加入 Supabase Data API 的 Exposed schemas。
2. 授予 `anon`、`authenticated`、`service_role` 對 `taipei_challenge` schema 的最小必要權限。
3. 前端查詢必須明確使用 `supabase.schema("taipei_challenge")`。
4. 前端環境變數只允許使用 `VITE_SUPABASE_URL` 與 `VITE_SUPABASE_ANON_KEY`。
5. `service_role key` 不得放入 GitHub Repository、GitHub Actions log、GitHub Pages 或任何前端程式碼。

## 5. 第 6 頁「與世界交朋友」內容設計

頁面名稱：

`與世界交朋友｜Connecting with the World`

活動方式：

1. 學生與外國遊客進行簡短英文訪談。
2. 頁面提供中英並陳的訪談範本文句。
3. 每一句英文提供「播放發音 Listen」功能。
4. 訪談後詢問外國朋友是否願意合照。
5. 合照後學生上傳照片。
6. 學生填寫外國朋友的國家，採自由輸入。
7. 不要求學生寫英文反思。

### 5.1 訪談句型

| 情境 | 中文 | English |
|---|---|---|
| 邀請訪談 | 你好，我們是大觀國中的學生。我們正在進行跟外國人交談的訪談活動。活動大概三到五分鐘，您願意接受我們的訪談嗎？ | Hello, we are students from Ta-guan Junior High School. We are doing an interview activity to talk with international visitors. It will take about three to five minutes. Would you be willing to answer a few questions? |
| 詢問國家 | 請問您是從哪一個國家來的？ | Which country are you from? |
| 詢問來臺原因 | 請問您為什麼會來臺灣？ | Why did you come to Taiwan? |
| 詢問喜歡之處 | 請問您最喜歡臺灣的哪一件事情？ | What do you like most about Taiwan? |
| 詢問不習慣之處 | 請問有沒有哪一個點，是您來臺灣最不喜歡或最不習慣的地方？ | Is there anything you dislike or are not used to in Taiwan? |
| 徵詢合照 | 請問我們可以和您一起拍一張合照嗎？ | May we take a photo with you? |
| 被拒絕時回應 | 沒關係。還是非常謝謝您！祝您有美好的一天！ | No problem. Thank you anyway! Have a nice day! |
| 訪談結束感謝 | 謝謝您接受我們的訪談。祝您在臺灣玩得愉快！ | Thank you for answering our questions. We hope you have a wonderful time in Taiwan! |

### 5.2 發音功能

每一句英文旁邊提供一個按鈕：

`播放發音 Listen`

規劃：

1. 使用瀏覽器內建語音功能播放英文。
2. 預設語音語言為英文。
3. 學生可重複播放，用於訪談前練習。
4. 若部分裝置不支援語音播放，仍顯示完整中英文句型，不影響任務完成。

### 5.3 學生作答欄位

| 欄位 | 類型 | 是否必填 |
|---|---|---|
| 外國朋友的國家 | 自由輸入 | 必填 |
| 合照照片 | 圖片上傳 | 必填 |
| 是否完成訪談 | 勾選 | 必填 |

### 5.4 隱私原則

第 6 頁不得蒐集：

1. 外國朋友姓名。
2. 電話。
3. Email。
4. 社群帳號。
5. 護照、地址或其他可識別個資。

## 6. 小組登入與教師後台登入

### 6.1 學生小組登入

學生不需要 Email。

建議流程：

1. 學生第一次進入網站，可自行建立隊伍。
2. 輸入隊名、通關碼、組員人數。
3. 系統依 3 至 6 人動態建立組員欄位。
4. 系統建立 anonymous user session。
5. 建立 `teams`、`team_members`、`team_sessions`。
6. 學生重新登入時，輸入隊伍代碼或隊名與通關碼。
7. `login_team()` function 驗證通關碼。
8. 驗證成功後，將目前使用者寫入 `team_sessions`。
9. RLS 依 `team_sessions.auth_user_id = auth.uid()` 判斷資料權限。

本次活動會先建立預設 3 組隊伍，但仍保留學生自行建立更多隊伍的功能。

### 6.2 教師後台登入

教師後台採兩層設計：

1. 共用入口密碼：`tkjhs810`。
2. Supabase Auth 教師帳號。

流程：

1. 教師進入後台入口。
2. 輸入共用入口密碼 `tkjhs810`。
3. 教師可註冊或登入 Supabase Auth。
4. 教師註冊不需管理者審核。
5. 註冊成功後建立 `teacher_profiles`，預設角色為 `teacher`。
6. 進入後台後，仍由 RLS 控制可讀寫資料。

說明：GitHub Pages 為靜態網站，前端程式碼可被檢查，因此 `tkjhs810` 只能作為入口門檻，不能作為唯一安全防線。

## 8. RLS 規劃

所有 `taipei_challenge` schema 內資料表都必須啟用 Row Level Security。

### 8.1 學生端權限

| 資料表 | 學生權限 |
|---|---|
| `activities` | 可讀取啟用中的活動 |
| `content_pages` | 可讀取啟用活動頁面 |
| `missions` | 可讀取啟用活動任務 |
| `teams` | 只能讀寫自己已登入的小組 |
| `team_members` | 只能讀寫自己小組成員 |
| `submissions` | 只能讀寫自己小組作答；送出鎖定後不可更新 |
| `world_friend_records` | 只能讀寫自己小組紀錄；送出鎖定後不可更新 |
| `media_files` | 只能讀寫自己小組媒體紀錄；送出鎖定後不可刪除 |

### 8.2 教師端權限

| 資料表 | 教師權限 |
|---|---|
| `activities` | 可讀取活動 |
| `content_pages` | 可讀取頁面內容 |
| `missions` | 可讀取任務內容 |
| `teams` | 可查看小組、隊名與通關碼 |
| `team_members` | 可查看組員 |
| `submissions` | 可查看作答 |
| `world_friend_records` | 可查看國家與合照紀錄 |
| `media_files` | 可查看媒體紀錄並取得播放或預覽資訊 |
| `audit_logs` | 一般教師不讀取，管理者可讀取 |

教師是否只能看負責小組，或可看本活動全部小組，可在初版實作時用 `assigned_teacher_id` 與角色控制。若活動現場老師需要互相支援，初版可允許 `teacher` 查看本活動全部小組。

### 8.3 管理者權限

管理者可讀寫 `taipei_challenge` schema 內本系統資料，但不得取得修改既有 `public` schema 的特殊授權。

管理者可：

1. 管理活動。
2. 管理任務內容。
3. 管理隊伍。
4. 查看全部成果。
5. 退回補件。
6. 匯出資料。
7. 調整教師角色。

## 9. Storage Policies 規劃

建立 private bucket：

`taipei-challenge-media`

檔案路徑格式：

`activity-id/team-id/mission-id/file-uuid.ext`

### 9.1 儲存內容

| 類型 | 說明 |
|---|---|
| 關卡照片 | 二二八和平公園、臺博館、臺北車站等照片 |
| 錄音 | 古生物館英文錄音 |
| 合照 | 第 6 頁與外國朋友合照 |

### 9.2 上傳規則

| 使用者 | 權限 |
|---|---|
| 學生 | 只能上傳到自己小組路徑 |
| 教師 | 初版不需要上傳學生媒體 |
| 管理者 | 可管理全部媒體 |

### 9.3 讀取規則

| 使用者 | 權限 |
|---|---|
| 學生 | 只能讀取自己小組媒體 |
| 教師 | 可讀取可見小組媒體 |
| 管理者 | 可讀取全部媒體 |

## 10. 自動儲存與離線策略

### 10.1 文字資料

1. 文字輸入先寫入瀏覽器本機資料。
2. 停止輸入約 1.5 至 2 秒後同步 Supabase。
3. 頁面顯示儲存狀態：
   - 正在儲存
   - 已儲存
   - 離線等待同步
   - 同步失敗
4. 重新整理、關閉頁面、短暫斷線後，不應遺失已填內容。

### 10.2 媒體資料

1. 照片上傳前壓縮，最長邊 1600 px。
2. 錄音建議每段不超過 2 分鐘。
3. 照片與錄音建立上傳佇列。
4. 網路中斷時保留待同步項目。
5. 離開頁面前若仍有未完成上傳，顯示提醒。

## 13. Rollback 規劃

rollback 文件會只移除本系統新增物件。

預計包含：

1. 刪除 `taipei_challenge` schema 內所有資料表、function、trigger、policy。
2. 刪除 `taipei-challenge-media` bucket 與 objects。
3. 刪除 `storage.objects` 上本系統專用 storage policies。
4. 不刪除既有 `public` schema。
5. 不刪除既有 Supabase Auth 使用者。
6. 不刪除其他網站既有 Storage bucket。

其中刪除 bucket 與 objects 屬高風險操作，實際執行 rollback 前需再次確認。
